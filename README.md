## 💡 Key Technical Features

### 1. Era-Aware Data Classification System

**Problem Solved:**

Powerball changed rules in 2015 (Powerball range: 1-35 → 1-26). Mixing historical data with current data causes:
- `IndexError: list index out of range` crashes
- Invalid frequency distributions
- Incorrect analytics results

**Solution:**

Dual-layer automatic classification:

```sql
-- Database Layer: SQLite triggers auto-classify
CREATE TRIGGER set_pb_era_on_insert
AFTER INSERT ON powerball_draws
FOR EACH ROW
BEGIN
    UPDATE powerball_draws
    SET 
        pb_is_current = CASE 
            WHEN NEW.pb BETWEEN 1 AND 26 THEN 1 
            ELSE 0 
        END,
        pb_era = CASE
            WHEN NEW.pb BETWEEN 1 AND 26 THEN '2015-now (1-26)'
            WHEN NEW.pb BETWEEN 27 AND 35 THEN '2012-2015 (1-35)'
            WHEN NEW.pb BETWEEN 36 AND 39 THEN '2009-2012 (1-39)'
            ELSE 'other'
        END
    WHERE rowid = NEW.rowid;
END;
```

```python
# Application Layer: Safety net
def update_pb_era_metadata():
    """Classify any untagged draws (belt-and-suspenders)"""
    conn.execute("""
        UPDATE powerball_draws
        SET pb_is_current = CASE WHEN pb BETWEEN 1 AND 26 THEN 1 ELSE 0 END
        WHERE pb_era = 'unknown'
    """)
```

**Result:**
- ✅ 1,678 current-era draws (used for analytics)
- ✅ 173 historical draws (preserved, not mixed)
- ✅ 100% classification accuracy
- ✅ Zero crashes in 6+ months production

**Adaptive Learning Process:**

```python
def update_strategy_weights():
    """Bayesian weight optimization based on ROI"""
    # 1. Calculate ROI for each strategy
    rois = {
        strategy: (prizes_won - cost_invested) / cost_invested
        for strategy in strategies
    }
    
    # 2. Normalize to weights (softmax-style)
    total_roi = sum(max(0, roi) for roi in rois.values())
    
    # 3. Update weights in database
    new_weights = {
        strategy: max(0, roi) / total_roi
        for strategy, roi in rois.items()
    }
    
    # 4. Best performers automatically get more tickets
    save_weights_to_db(new_weights)
```
